# Platform Documentation

The FOS platform consists of three main frontend applications designed for different user journeys.

## Applications

| Application | Description |
|-------------|-------------|
| [Landing Website](#landing-website) | Public-facing site for marketing, whitelist registration, and minting |
| [Authentication](#authentication) | Wallet-based auth with SIWE (Sign-In with Ethereum) |
| [Dashboard](#dashboard) | User portfolio, mint history, and claim management |
| [Admin Panel](#admin-panel) | Role-gated controls for phases, parameters, and operations |

## Tech Stack

| Layer | Technology | Purpose |
|-------|------------|---------|
| **Framework** | Next.js 15 | React with SSR/SSG |
| **Styling** | TailwindCSS | Utility-first CSS |
| **Web3** | wagmi + viem | Wallet interactions |
| **Auth** | SIWE + Iron Session | Secure authentication |
| **State** | Zustand | Client state management |

### Wallet Integration

```typescript
import { useConnect, useAccount } from 'wagmi';
import { InjectedConnector } from 'wagmi/connectors/injected';

function ConnectButton() {
  const { address, isConnected } = useAccount();
  const { connect } = useConnect({
    connector: new InjectedConnector(),
  });

  if (isConnected) {
    return <span>Connected: {address}</span>;
  }

  return (
    <button onClick={() => connect()}>
      Connect Wallet
    </button>
  );
}
```

## Authentication

### Authentication Flow

1. **Connect Wallet** - User connects via MetaMask, WalletConnect, or other wallet
2. **Sign Message** - Server generates SIWE message, user signs with wallet
3. **Verify Signature** - Backend verifies signature and creates session
4. **Session Management** - Encrypted cookie maintains auth state

### SIWE Implementation

```typescript
import { SiweMessage } from 'siwe';

// Generate message
const message = new SiweMessage({
  domain: window.location.host,
  address: userAddress,
  statement: 'Sign in to FOS Platform',
  uri: window.location.origin,
  version: '1',
  chainId: 1,
  nonce: await fetchNonce(),
});

// User signs
const signature = await wallet.signMessage(message.prepareMessage());

// Verify on backend
const verified = await message.verify({ signature });
```

## Admin Panel

> **Warning**: Admin access requires the `ADMIN_ROLE` on-chain permission.

### Capabilities

- **Phase Control**: Enable/disable mint phases
- **Parameter Updates**: Modify prices, limits, windows
- **Root Updates**: Push new Merkle roots
- **Emergency**: Pause contracts, emergency withdrawals
